# Desafios de Programação

Este diretório contém desafios de programação para membros do time em todos os níveis de experiência.

---

## Índice de Desafios

### Iniciante

Adequado para desenvolvedores que são novos no time ou que estão revisando conceitos fundamentais. Cada desafio deve levar menos de uma hora.

| # | Arquivo | Tópico |
|---|---|---|
| 01 | [`beginner_challenge_01.md`](beginner_challenge_01.md) | FizzBuzz com um Toque — loops, condicionais, funções |
| 02 | [`beginner_challenge_02.md`](beginner_challenge_02.md) | Conversor de Temperatura — POO, validação de entrada, tratamento de erros |

Adequado para desenvolvedores com alguma experiência que estão prontos para enfrentar problemas mais realistas. Cada desafio deve levar de 2 a 4 horas.

| # | Arquivo | Tópico |
|---|---|---|
| 01 | [`intermediate_challenge_01.md`](intermediate_challenge_01.md) | Parser de Pacotes de Telemetria — protocolos binários, POO, tratamento de erros |
| 02 | [`intermediate_challenge_02.md`](intermediate_challenge_02.md) | Máquina de Estado de Voo — design de FSM, programação orientada a eventos |

## Como Tentar um Desafio

1. Leia o documento do desafio completamente antes de escrever qualquer código.
2. Crie um branch seguindo a convenção de nomenclatura nas instruções do desafio.
3. Escreva sua solução, depois escreva seus testes.
4. Execute o lint e os testes localmente antes de submeter.
5. Envie via pull request conforme descrito em cada desafio.

Veja [`activities/submissions/README.md`](../submissions/README.md) para detalhes de submissão.
